Command Injections

Created2022-09-16|Updated2022-09-16|HTB AcademyBug Bounty Hunter
|Post View:

Tespit

Bu tarz alanlarda komut enjeksiyonu olabilir. Örneğin yukarıdaki işlem IP adresi girmemizi istiyor ve host’u check ediyor. Yani oraya bir ping yaptığını söyleyebiliriz. Ping’in bir OS komutu olduğunu düşünürsek, eğer burada yeterli önlem alınmadıysa o zaman buraya kendi komutlarımızı yerleştirebiliriz. İşte buna command injection denir.

Kendi komutumuzu eklemek için bir önceki komutu bir nevi noktalamamız gerekir. Bunun için aşağıdaki tablodaki karakterlerden yararlanabiliriz:

Injection Operator Injection Character URL-Encoded Character Executed Command
Semicolon ; %3b Both
New Line \n %0a Both
Background & %26 Both (second output generally shown first)
Pipe %7c
AND && %26%26 Both (only if first succeeds)
OR
Sub-Shell `` %60%60 Both (Linux-only)
Sub-Shell $() %24%28%29 Both (Linux-only)

Bu operatörlerden herhangi birini başka bir komut enjekte etmek için kullanabiliriz, böylece komutlardan ikisi (daha önceki yazılan ve bizim şimdi yazdığımız) veya herhangi biri yürütülür.

🌐 Tip: In addition to the above, there are a few unix-only operators, that would work on Linux and macOS, but would not work on Windows, such as wrapping our injected command with double backticks (````) or with a sub-shell operator ($()).

Genel olarak, temel komut enjeksiyonu için, web uygulaması dili, framework’ü veya back-end sunucusundan bağımsız olarak bu operatörlerin tümü komut enjeksiyonları için kullanılabilir. Bu nedenle, ister Linux sunucusunda çalışan bir PHP web uygulamasına, ister bir Windows arka uç sunucusunda çalışan bir .Net web uygulamasına ya da ister macOS arka uç sunucusunda çalışan bir NodeJS web uygulamasına enjekte ediyor olalım, ne olursa olsun enjeksiyonlarımız çalışmalıdır.

🌐 Note: The only exception may be the semi-colon ;, which will not work if the command was being executed with Windows Command Line (CMD), but would still work if it was being executed with Windows PowerShell.

Peki bunu burp suite kullanırken payloadlarımızda kullanabilir miyiz? Evet! Direkt olarak yukarıda yazdığımız şekilde ancak echo barındırmayacak haliyle payloadımıza ekleriz. Şimdi 127.0.0.1${LS_COLORS:10:1}${IFS} şeklindeki yeni payloadımıza bir göz atalım.

Character Shifting

Filtrelemeler ve kara listelerden kurtulmak için bir başka teknik de karakter değiştirme yöntemidir. Bu sefer burada devreye ASCII karakterleri giriyor. Bize lazım olan karakterin ASCII değerinin bir eksiğindeki karakteri alıyoruz ve onun değerini shift’leyip (bir nevi onun değerini ittirip) onu bizim istediğimiz yeni karaktere dönüştürüyoruz. Örneğin bize \ karakteri lazım ve bu karakterin ASCII değeri 92. Bu doğrultuda biz ASCII değeri 91 olan karakteri gidip buluyoruz ve bu karakter de [ ’dir. Sonrasında:

🌐 Çok Önemli Not: Mutlaka ama mutlaka payload yazarken yeni satıra geç. Yani new-line, %0a karakterini kullan. Yoksa dönen cevabı göremeyebilirsin. Bunun bir örneği aşağıda yaşandı. %0a kullanmadan önce sadece ping yanıtını görebiliyordum. %0a kullandıktan sonra artık cevabı görebilir hale geldim.
