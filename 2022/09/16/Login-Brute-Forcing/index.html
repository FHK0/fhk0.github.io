User:Pass Kombinasyonuna Brute Force

Bazen sitelerde default olan kullanıcı adı - şifre kombinasyonları vardır. Örneğin kullanıcı adı: admin şifre: 123456 şeklinde. Bu tarz durumları test etmek için hydra kullanarak denemeler yapabiliriz. aşağıdaki tablo kullanacağımız parametreleri göstermekte:

Options Description
-C ftp-betterdefaultpasslist.txt Combined Credentials Wordlist
SERVER_IP Target IP
-s PORT Target Port
http-get Request Method
/ Target Path

Kullanıcı Adı / Şifrelere Saldırı

Hydra’nın bir hedefe saldırabilmesi için ona belirli kullanıcı adı ve şifreleri içeren wordlist’ler vermeliyiz. Bu belirtmeleri yapmak için kullandığımız bazı flag’ler vardır.

  • -P: Parola wordlist’ini belirtmekte kullanılır.
  • -L: Kullanıcı adı wordlist’ini belirtmekte kullanılır.
  • -f: Başarıyla tamamlanan ilk kullanıcı adı - şifre kombinasyonu girişinden sonra işlemi durdurması gerektiğini bildiren flag.
  • -u: Bir parolayı tüm kullanıcılarda dener. Eğer bu flag kullanılmazsa bir kullanıcıda parola wordlist’indeki tüm parolalar denenir ve tüm parolalar denendikten sonra ikinci kullanıcıya geçer ve onda da tüm parolaları dener. Biz bunu her zaman tercih etmeyebiliyoruz.

Burada,

  1. http[s]-{head|get|post}
  2. http[s]-post-form

http modülleri ilgimizi çekmektedir. İlk sıradaki az önce örneğini yaptığımız HTTP doğrulamasını kapsarken ikincisi ise .php .aspx sayfalarında bulunan, hepimizin aşina oldu login form’larını kapsar.

Peki sitedeki login form’u GET form mudur yoksa POST form mudur bunu nasıl anlarız? Eğer form’a girdiğimiz input daha sonra gönderim yapılırken URL’de gözüküyorsa bu GET form’dur. Aksi takdirde, URL üzerinde herhangi bir değişiklik olmuyorsa bu POST form’dur. Bu farkın ayırdında olarak isteklerimizi o şekilde düzenleyebiliriz.

Sonuç:

Özetle üç şey gerekiyor:

  • URL patikası

/login.php

  • POST parametreleri

/login.php:[user parameter]=^USER^&[password parameter]=^PASS^

  • Başarısız veya başarılı olunduğunda alınması beklenen string mesajı

/login.php:[user parameter]=^USER^&[password parameter]=^PASS^:[FAIL/SUCCESS]=[success/failed string]

Fail/Success String

Fail string’i sağlarsak, yanıtta string bulunmayana kadar aramaya devam eder. Başka bir yol, bir başarı string’i sağlarsak, yanıtta string bulunana kadar aramaya devam eder.

Type Boolean Value Flag
Fail FALSE F=html_content
Success TRUE S=html_content
