Absent Validation

Herhangi bir filtrelemenin olmadığı ve dosya yükleme noktasına bu eksiklikten mütevellit istediğimiz dosyaları yükleyebildiğimiz güvenlik düzeyidir. Yani herhangi bir güvenlik dahi yoktur. Genellikle buralarda ya web shell kullanırız ya da reverse shell alırız.

🌐 Web Shell Nedir?: A Web Shell provides us with an easy method to interact with the back-end server by accepting shell commands and printing their output back to us within the web browser. A web shell has to be written in the same programming language that runs the web server, as it runs platform-specific functions and commands to execute system commands on the back-end server, making web shells non-cross-platform scripts. So, the first step would be to identify what language runs the web application.

Shell atabilmemiz için karşımızdaki sayfanın hangi uzantıya sahip olduğunu bilmeliyiz. Yani bu sayfa .html mi .php mi .asp mi ona göre scriptimizi yazacağız. Bunu belirlemek için fuzz’lama yapabiliriz, index.html-index.php-index.asp şeklinde denemeler gerçekleştirebiliriz ya da wappalyzer gibi uygulamalar-eklentiler kullanabiliriz.

Filtrelemeleri Atlatmak

Çoğu filtreleme front-end tarafında yapıldığı için bu filtrelemeleri atlatabiliriz. Bunu antlamanın basit yolu dosyamızı seçtikten sonra sayfa hiçbir zaman yenilenmezse veya herhangi bir HTTP isteği göndermezse o zaman işlemler front-end tarafta oluyor demektir.

Any code that runs on the client-side is under our control. 😈

Özetle, filtreleri atlatmak için ya giden HTTP isteğini manipüle edeceğiz ya da front-end taraftaki kodu devre dışı bırakacağız. Şimdi bu konuları inceleyelim.

Back-end Request Modification

HTTP isteklerini manipüle ederken ne kullanıyorduk? Tabii ki Burp Suite.

Dosya yüklerken bir POST isteği gönderiyoruz ve bu POST isteği içerisinde dosya adımız ve dosyanın içeriği de bulunmakta. Dosya/resim yüklemesi yaparken buradaki içerikleri manipüle ederek kendi shell’imizi yükleyebiliriz.

Dosya adını uzantısı uzantımıza uyacak şekilde değiştirip içeriği de shell kodu olarak değiştirirsek kendi shell’imizi karşı tarafa yüklemiş oluruz. Ardından dosyayı yüklendiği yerde ziyaret ederek remote code execution elde edebiliriz.

Type Filters

Content-Type

Tip filtrelemesi bu zamana kadar öğrendiğimiz tüm yöntemleri boşa çıkarabilir. Çünkü giden HTTP isteğinde dosyamızın ne tür bir içerik türüne sahip olduğu belirtilir ve bu unsur filtrelemeden geçiriliyorsa o biz ne kadar dosya uzantısını değiştirirsek değiştirelim yine de işe yaramaz. O yüzden yapılacak iş HTTP isteğini düzenlemektir. Örnek bir php kodu inceleyerek bu content-type başlığının nasıl filtrelendiğini inceleyelim:

Web tarayıcılarımız tarafından dosya yüklenirken bu tip otomatik olarak belirlense de bu işlem bir kullanıcı tarafı işlemidir yani bizim kontrolümüz altındadır. Tip fuzz’laması yaparak hangi içerik tipinin onaylanıp hangilerinin onaylanmayacağını belirleyebiliriz. 🌐 Note: A file upload HTTP request has two Content-Type headers, one for the attached file (at the bottom), and one for the full request (at the top). We usually need to modify the file’s Content-Type header, but in some cases the request will only contain the main Content-Type header (e.g. if the uploaded content was sent as POST data), in which case we will need to modify the main Content-Type header.

MIME Type

MIME Tipi dosyanın hangi unsurla başladığına bakar. Örneğin eğer düz yazı ile başlıyorsa bu bir txt dosyasıdır. Ya da bir GIF dosyası ise en başında GIF98 gibi o dosyanın ne tür bir unsur olduğunu tanımlayan bir ibare bulunur. Dosyamızın adı shell.php olsa da başlangıcında GIF98 yazdığı için karşı sitedeki filtre bunu bir gif dosyası zanneder ve o dosyayı kabul eder.

https://en.wikipedia.org/wiki/List_of_file_signatures

Yukarıdaki link üzerinden dosya türlerinin nasıl bir MIME tipine sahip olduğu öğrenilebilir. Bu MIME tipini değiştirerek o dosyanın algılanma biçimini değiştirebiliriz ve bu sayede filtreleri atlatabiliriz. Web serverlar daha kesin sonuçlar verdiği için .jpg gibi uzantılara bakmak yerine bu MIME tipi filtrelemesini uygularlar.
