Lab: Username enumeration via different responses

To solve this lab, we will simply perform a brute force attack. First we need to set the username. For this, let’s take a look at the error that occurs when we enter the username incorrectly. The login page returns us an error as Invalid username. We try among the candidates and whichever we don’t see this error, that value is our username.

As seen above, our code worked and told us that our username is analyzer. Now it’s time to find the password.

In order to find the password, let’s first take a look at how the system reacts with the correct username and wrong password combination. The system returns an error as Incorrect password. When we do not get this error, then we have found the password.

As can be seen in the picture above, we saw that the jessica password was working as a result of the brute force attack, and when we logged into the system with this information, we solved the lab.

The Logic Behind the Solution

All the logic here is based on the answers returned through the system. While testing candidate usernames and passwords, we determined which username and password were correct in line with the system’s responses to us. We performed these operations with python and its requests module. We also used the sys module and made the code work via terminal.
