Lab: SQL injection UNION attack, determining the number of columns returned by the query

To solve this lab, we need to find the number of columns returned by the query. We will use the NULL value for this. We know that the category parameter is available for injection. So we will inject into this parameter.

As seen in the picture above, we have chosen any category and now let’s do our first try here.

If we encounter an error like the one above, it indicates that we could not find the column number of the sql query. Whenever we do not encounter a server error, then we can say that there are as many columns as NULLs that we have written.

Finally, we reached the result with three NULL values that we wrote. This means that the query consists of three columns.

The Logic Behind the Solution

NULL is convertible to every commonly used data type, using NULL maximizes the chance that the payload will succeed when the column count is correct. The NULL value corresponds to each column in the query, and there should be as many NULL values as there are columns. Thus, when the NULL value does not give an error, we find the number of columns.
