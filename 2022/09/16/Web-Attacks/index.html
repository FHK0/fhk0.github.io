Burada file_id=123 dediğimiz zaman rahatça gidebiliyorsak o zaman id’si 122 olana da 121 olana da hatta belki de id değeri 500 olana rahatça gidebiliriz. Tek yapmamız gereken sayılarla oynamak. Bunun önüne geçmek için çeşitli şifrelemeler ve hash’ler kullanılıyor. IDOR tespiti için Javascript içerisindeki AJAX çağrıları kullanılabilir. Bypassing Encoded References

Bazı dosya isimleri şifrelenmiş ya da hash’lenmiş olabilir. Örneğin bir sözleşme dosyası şöyle bir isme sahip olabilir:

Eğer bu hashleme işlemi front-end tarafta yapılıyorsa bu durumda hangi metotların kullanıldığını öğrenebilir ve durumu tersine çevirebiliriz.

Yukarıda uid değeri sözleşmenin değeri olarak kullanılmış. İçeride de btoa ve md5 görülmekte. Burada btoa base 64 şifrelemesini temsil etmekte. Yani uid değeri önce base 64’e sonra md5’e çevrilmiş vaziyettedir. Örneğin id değeri olarak 1 verelim ve yukarıdaki contract ile eşleşip eşleşmediğini görelim.

🌐 Tip: We are using the -n flag with echo, and the -w 0 flag with base64, to avoid adding newlines, in order to be able to calculate the md5 hash of the same value, without hashing newlines, as that would change the final md5 hash.

🌐 Note: Some web applications may default to a JSON format in HTTP request, but may still accept other formats, including XML. Advanced File Disclosure

CDATA kullanarak veri çekelim. Eğer çıktı formalı XML’e uymuyorsa CDATA ile dosyayı derleriz
(<![CDATA[ FILE_CONTENT ]]>) ve bu şekilde XML ayrıştırıcısı, herhangi bir özel karakter de dahil olmak üzere herhangi bir veri türünü içerebilen bu kısmı ham veri (raw data) olarak değerlendirecektir. Bu sayede herhangi bir karakter engeline takılmadan dosyayı çekebileceğiz.

XML dahili ve harici varlıkların birleştirilmesini engeller. Bu yüzden XML Parameter Entities ‘lerden faydalanacağız. Bunlar % karakteriyle başlayan ve yalnızca DTD içinde kullanılabilen özel bir varlık türüdür.
